## :triangular_flag_on_post: Level2-Level3
The password for the next level is stored in a file called "**--spaces in this filename--**" located in the home directory
> **Given Information**
  1. username: bandit2
  2. password: from the previous level 
  3. SSH port: 2220
  4. Host: bandit.labs.overthewire.org

> **Solution**
- We log into the game using SSH by typing this command `ssh bandit2@bandit.labs.overthewire.org -p 2220`.
- After login we take a look at the content of our working directory/home directory. We find a file name with spaces and double dashes in both ends
- Normally, we could just open a file by typing a `cd filename`, but however, the file name is spaced so it won't play very well with the terminal. Plus there is another trap with them double dashes
- In that case we could enclose our file name with quotation marks or hitting `TAB` the moment we start writting the initials of our file name
- So, to open a file we type `cat ./"--spaces in this filename--"` or `cat ./--spaces in this filename--`

> **Password**
- **MN*************lSmx**

> **Moral of the Story**
- FIrst, the `TAB` on your keyboard is your best friend when using the terminal. For autocompletion
- Second, remember the thingy I said about the double dashes`--`? Well, those double dashes are standard convention to express option of a command to the argument that follows. Example `apt list --installed`. Pretty cheeky!
- Quotations, can also deal with tricky filenames.
- There is a way to write spaced filenames. it involves `\`
